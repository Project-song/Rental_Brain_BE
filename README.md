# Rental Brain  
### CRM 기반 B2B 렌탈 운영 관리 시스템

> **“렌탈 운영을 ‘관리’가 아니라 ‘예측’과 ‘대응’의 문제로 재정의하다.”**

Rental Brain은 B2B 렌탈 기업이 겪는  
**계약 관리 · 자산 운영 · 연체 대응 · 고객 이탈 위험** 문제를  
CRM 관점에서 통합 관리하기 위해 설계된 **렌탈 운영 ERP 시스템**입니다.

단순 CRUD를 넘어,  
**고객 상태를 세그먼트로 정의하고 → 자동 액션과 KPI로 연결**하는 것을 목표로 합니다.

---

## 1. 프로젝트 배경 & 문제 인식 (Needs Analysis)

### 기존 렌탈 운영의 한계
- 엑셀/수기 중심 운영으로 인한 데이터 단절
- 계약·자산·연체·AS·피드백 정보의 분산
- 재계약/이탈 위험을 사후에 인지
- 담당자 경험에 의존한 운영 판단

### 해결하고자 한 핵심 질문
- 이 고객은 **지금 어떤 상태인가?**
- 위험 신호를 **사전에 감지**할 수는 없을까?
- 운영 액션을 **데이터 기반으로 연결**할 수는 없을까?

---

## 2. 해결 전략 & 시스템 컨셉 (Analysis & Design)

### 핵심 컨셉
**Segment-driven Rental CRM**

> 고객을 ‘분류’하지 않고  
> **상태(Segment)로 정의하여 액션과 KPI로 연결**

### 고객 관리 여정 (Customer Journey)
> 수정해야할 것, 다이어그램이나 삽도 집어 넣고 작성하기

### 설계 원칙
- 세그먼트는 고객 **상태 기반 액션 트리거**
- 고객은 (초기 기준) **동시에 하나의 세그먼트만 보유**
- 이탈은 세그먼트가 아닌 **결과 지표(KPI)**
- 조회(MyBatis) / 명령(JPA) 분리
- 모든 상태 변화는 **히스토리 & 스냅샷으로 추적**

---

## 3. 주요 기능 (Core Features)

### 고객 & 세그먼트 관리
- 고객 상태 자동 판별
- 세그먼트 전이 히스토리 관리
- 이탈 위험 고객 탐지

### 계약 & 재계약 관리
- 계약 / 재계약 프로세스 관리
- 재계약 타이밍 기반 고객 분류

### 자산 & 연체 관리
- 자산 대여 / 회수 / 수리 이력 관리
- 연체 상태 감지 및 리스크 연계

### 액션 (프로모션 / 쿠폰)
- 세그먼트 기반 쿠폰·보상 정책
- 고객 상태별 운영 액션 연결

### KPI & 대시보드
- 전체 이탈 위험률
- 세그먼트별 이탈 가능성
- 위험 사유 분포 및 추이

---

## 4. 기술 스택 (Tech Stack)

### Backend
- Java 17
- Spring Boot
- Spring Data JPA (CUD)
- MyBatis (조회 전용)
- MariaDB
- Swagger / OpenAPI

### Frontend
- Vue 3
- Vite
- Pinia
- ECharts

---

## 5. 구현 포인트 (Implementation Highlights)

### MyBatis + JPA 분리 전략
- 복잡한 KPI/통계 조회는 **MyBatis**
- 도메인 상태 변경은 **JPA**
- 성능과 도메인 일관성 확보

### 세그먼트 & 리스크 스냅샷
- 월 단위 고객 상태 스냅샷 저장
- 리스크 전이 히스토리 관리
- 위험 발생 사유 추적 가능

### 코드 체계 & 데이터 정합성
- `id_sequence` 기반 코드 자동 생성
- 실제 기업명 기반 1글자 변형 더미데이터
- FK / Check 제약조건으로 데이터 무결성 유지

## 8. 프로젝트를 통해 얻은 것

- 업무 흐름 중심의 도메인 설계 경험
- CRM 관점에서 **데이터 → 상태 → 액션 → KPI**를 연결하는 구조 설계
- 단기 기능 구현이 아닌 **지속 가능한 운영 구조**에 대한 고민
